# Week 2 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.  
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** basic 10개 + 난이도 하 6개 + 난이도 중 6개 = 총 22개 과제 중 **22개 해결** (100%)
- **달성률 산식:** (해결한 과제 수 / 22) × 100


## 2. 설계
- **목표:** 재귀·백트래킹 관련 6개 과제(팩토리얼, 피보나치, 하노이 탑, 조합 생성, 외판원 순회 2, N-Queen)에서 **코드 작성 전 흐름/구조를 문서/주석으로 설계** 후 구현
- **달성률 산식:** (설계 후 구현 완료한 과제 수 / 6) × 100


## 3. 구현
- **목표:** basic 10개 테스트 전체 통과 + 난이도 하·중 12개 문제 풀이 = **22개 구현 완료**
- **달성률 산식:** (테스트 통과 + AC 채점 완료한 문제 수 / 22) × 100


## 4. 품질
- **목표:** 풀이한 **22개 과제 모두** 백준/LeetCode 예제·테스트 케이스 통과
- **달성률 산식:** (통과한 과제 수 / 22) × 100


## 5. 유지보수
- **목표:** 이번 주(토~목) **6일 중 5일 이상** 개인 레포지토리에 의미 있는 커밋 메시지로 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 공통 2개(팀 협업 룰/팁 정리·합의) + 수요 코딩회 팀 활동 참여 = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 수요 코딩회 앱에 대한 **사용자 가치·방향성**을 README 등에 1문장 이상 명시
- **달성률 산식:** 명시 여부 (0% / 100%)


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + AI 생성 코드 검증·원리 설명(수요 코딩회 발표 시 1회 이상) = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 10. 학습 민첩성
- **목표:** basic 10개 주제(리스트·딕셔너리, 배열, 문자열, 완전탐색, 재귀, 백트래킹, 복잡도, 버블 정렬, 삽입 정렬, 정수론) 중 **10개 모두** 새로 학습 후 적용
- **달성률 산식:** (학습 후 적용 완료한 주제 수 / 10) × 100

### 달성률 산식 참고
- 문제해결/구현/품질: 과제·문제 단위로 카운트
- 유지보수: 일 단위 커밋 빈도
- 협업/태도/AI 활용: 완료 항목 수
- 비즈니스 이해: 명시 여부 (예/아니오)
