## 1. 이번 주 목표

## 2. 시도한 접근 방식
- basic 빈 코드 채우기로 주요 개념(배열, 문자열, 재귀, 백트래킹, 정렬, 정수론)을 먼저 학습한 뒤, 난이도 하·중 문제에 적용
- 재귀·백트래킹 문제(하노이 탑, 외판원, N-Queen 등)는 코드 작성 전 흐름을 설계하고 구현
- 수요 코딩회에서는 OpenAI 코딩 워크샵 기반으로 팀 프로젝트를 진행하고 목요일에 개선 발표


## 3. 문제와 해결 과정

### basic 품질 검증 한계
- **문제:** basic 10개 과제는 백준 링크가 없어 채점 확인이 불가
- **해결:** 백준 링크가 없는 basic 문제는 로컬 테스트 통과 여부로 품질을 측정하는 것으로 기준을 변경

### 재귀·백트래킹 설계의 중요성
- **문제:** 하노이 탑, N-Queen 같은 문제에서 바로 코드를 작성하면 흐름이 꼬여 디버깅이 어려움. 또한 N-Queen 문제에서 비효율적인 설계로 백준 테스트 케이스 통과가 실패하였음.
- **해결:** 재귀 호출 구조와 탈출 조건을 먼저 그림/주석으로 설계한 뒤 구현. gpt로부터 작성한 N-Queen 문제 풀이코드의 문제에 대한 리뷰를 받은 뒤 힌트를 얻어 풀이 실시.

### AI 활용 원칙 실천 부족
- **문제:** AI 활용 원칙은 수립했으나, AI 코드 검증·원리 설명을 실제로 실천하지 못함 (달성률 50%)
- **개선:** 다음 주부터 AI가 생성한 코드를 사용할 때 반드시 기술적 원리를 익히고 정리하여 팀원에게 설명하는 과정을 추가

### CODEX 앱 사용시 프롬프트 개선 부족
- **문제:** 설계 문서를 먼저 작성하였고 codex 앱에서 AI에게 설계 단계별로 앱의 폼을 만들라고 지시하였으나, AI 에이전트의 동작을 통제하지 못함.
- **개선:** CODEX 앱이 아닌 커서를 통해 AI 행동지침에 대한 프롬프트를 먼저 작성하고, AI 에이전트의 동작을 모듈 단위로 하나씩 제어

## 4. 새로 배운 점
- 완전탐색·재귀·백트래킹: 문제 유형별 접근 전략과 설계 → 구현 흐름
- 정렬(버블·삽입): 파이썬 내장 sort()만 쓰다가, 직접 구현하니 데이터 크기에 따른 성능 차이를 체감
- 정수론(GCD/LCM, 소수, 골드바흐): 유클리드 호제법, 에라토스테네스의 체
- GitHub Projects: 칸반 보드 기반 진척도 관리
- CODEX 워크샵: 시스템 설계 문서 없이 요구사항만으로 Codex에 '멀티 프롬프트 생성기'를 만들게 함. 하나의 index 파일에 모든 로직이 들어가 구조는 이상하나 MVP는 동작
- 수요 코딩회: '세계 이슈 맵'(세계의 전쟁·경제·정치·자연재해 기사를 수집하여 지도에 위치 맵핑) 개발. Cursor로 기획부터 모듈 단위 코드 작성까지 세밀하게 제어하며 진행
- 퀴즈: 6문제 중 5문제를 틀림. 파이썬 자료구조의 mutable/immutable 개념을 제대로 이해하지 못한 것이 원인. 오답을 통해 리스트(mutable)와 정수, 튜플, 문자열(immutable)의 동작 차이를 깊이 공부할 수 있는 기회가 됨
