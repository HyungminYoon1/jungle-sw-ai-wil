## 2. 시도한 접근 방식

- AI는 기획·설계·분석에만 사용해야 하나, 시간 내 완성을 위해 AI의 도움을 받아 코드도 작성


## 3. 문제와 해결 방법

### 다수 사용자 접속 시 부하 완화
- **문제:** 보드에 여러 사용자가 접속했을 때 실시간 업데이트 방식은 드래그 중 좌표를 계속 전송해 부하가 큼
- **해결:** 실시간 전송 대신, 포스트잇 이동 완료 시점에 최종 좌표만 백엔드로 전송하는 방식으로 변경하여 요청 빈도를 줄임

### S3 이미지가 스냅샷에 보이지 않음
- **문제:** HTTP로 접속 시 보드 스냅샷을 찍으면 S3에 저장된 이미지가 스냅샷에 표시되지 않음
- **해결:** 도메인을 HTTPS로 변경하니 스냅샷 캡처 시 이미지가 정상적으로 포함됨


## 4. 새로 배운 점

- Flask + Jinja2 SSR: 템플릿으로 페이지 구조를 관리하고 서버에서 렌더링
- 팀 협업: Notion 에서 업무 정의 및 역할 분담, 일정 관리, 동료 코드 리뷰 경험
- AI 도구를 적극 활용하여 프로젝트를 주어진 시간 내에 완성도 높게 구현할 수 있었으나, 개개인이 직면한 문제를 깊이있게 파고들어 몰입하며 학습하는 경험을 놓침
