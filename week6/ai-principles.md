## 핵심 원칙

- AI는 학습 파트너이며, 결과보다 이해·검증·설명 가능 여부가 더 중요함
- 퀴즈나 제한 환경 과제에서는 AI와 인터넷을 사용하지 않음
- 학습 과제에서 AI를 사용할 때는 포인터 흐름, 메모리 할당/해제, 자료구조 동작 원리를 직접 설명할 수 있어야 함
- 수요 코딩회에서는 AI를 적극 활용하되, 생성된 코드의 핵심 로직과 한계를 README 발표 수준으로 설명할 수 있어야 함
- INSERT, SELECT, SQL 파싱, 실행 분기, 파일 저장 포맷은 예제 입력과 테스트 케이스로 직접 검증하기
- AI가 생성한 코드는 바로 제출하거나 머지하지 않고, 입력 케이스와 예외 케이스를 확인한 뒤 사용하기
