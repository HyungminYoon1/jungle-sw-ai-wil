# Week 6 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.  
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** C 학습 핵심 주제 4개(`컴파일/디버깅/make`, `포인터`, `malloc/free`, `자료구조 동작 원리`) + SQL 처리 흐름 4개(`입력`, `파싱`, `실행`, `저장`) = 총 **8개 이상 문제 해결**
- **달성률 산식:** (해결한 핵심 주제 수 / 8) × 100


## 2. 설계
- **목표:** 수요 코딩회 SQL 처리기 구현 시 **CLI 입력 흐름, SQL 파싱 구조, 실행 계층 분리, 파일 저장 포맷** 등 설계 포인트 **4개 이상**을 README 또는 설계 메모에 명시
- **달성률 산식:** (문서에 명시한 설계 포인트 수 / 4) × 100


## 3. 구현
- **목표:** Data Structures 구현 27개 완료 + SQL 처리기 핵심 기능 5개(`CLI`, `INSERT`, `SELECT`, `파일 저장`, `파싱`) 구현 = **32개 이상 구현 완료**
- **달성률 산식:** (완료한 구현 항목 수 / 32) × 100


## 4. 품질
- **목표:** Data Structures 4개 영역(`Linked List`, `Stack/Queue`, `Binary Tree`, `BST`) 직접 테스트 + SQL 처리기 단위/기능/엣지 테스트 3개 = **7개 이상 검증 항목 통과**
- **달성률 산식:** (통과한 검증 항목 수 / 7) × 100


## 5. 유지보수
- **목표:** 이번 주(금~목) **7일 중 5일 이상** 개인 레포지토리에 의미 있는 커밋 메시지로 C 학습 내용, 자료구조 구현, SQL 처리기 관련 작업을 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 팀 협업 룰/팁 합의(1개) + Data Structures 개념 공유 및 상호 점검(1개) + 수요 코딩회 SQL 처리기 팀 프로젝트 참여(1개) = **3개 완료**
- **달성률 산식:** (완료한 항목 수 / 3) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 이번 주에 구현한 SQL 처리기가 **왜 필요한지**, 그리고 일반적인 DBMS와 어떤 차이가 있는지를 README에 **2문장 이상** 명시
- **달성률 산식:** (명시한 문장 수 / 2) × 100


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + SQL 처리기 또는 자료구조 구현 코드의 핵심 로직 검증·원리 설명(1개) = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 10. 학습 민첩성
- **목표:** `C 문법`, `포인터`, `malloc/free`, `Linked List`, `Stack/Queue`, `Binary Tree`, `BST`, `SQL 파싱/실행 흐름`을 포함해 **8개 이상 개념/구현 포인트**를 학습 후 적용
- **달성률 산식:** (학습 후 적용 완료한 개념/구현 포인트 수 / 8) × 100
