## 1. 이번 주 목표
- C언어 문법을 자료구조 문제 풀이와 SQL 처리기 과제를 수행하는 데 필요한 수준까지 이해하기
- 포인터를 단순 문법이 아니라 `값`, `주소`, `역참조`, `함수 인자 전달` 관점에서 이해하기
- 자료구조 문제에서 제공된 스켈레톤 코드의 일부 함수를 구현하며 C언어와 포인터 사용에 익숙해지기
- SQL 처리기의 전체 구조를 `입력 -> 파싱 -> 실행 -> 저장` 흐름으로 설명할 수 있을 정도로 이해하기
- Linux/gcc 사용법과 `malloc/free`는 과제 진행에 필요한 최소 수준만 확인하고, 핵심 학습 시간은 C언어와 포인터 이해에 더 많이 쓰기
- CS:APP 3장을 읽으며 C 코드가 실제로 어떻게 실행되는지 흐름을 이해해 보기

## 2. 시도한 접근 방식
- 먼저 `week6/learning/c-language-basics.md`를 정리하면서 변수, 함수, 구조체, 포인터를 한 흐름으로 다시 봤다.
- Linux/gcc 사용법과 `malloc/free`는 자료구조 과제를 해결할 수 있는 정도만 간단하게 확인했다.
- 자료구조 파트는 `Linked List -> Stack/Queue -> Binary Tree -> BST` 순서로 문제를 보면서, 제공된 C 스켈레톤 코드 안에서 빈 함수를 채우는 방식으로 접근했다.
- 연결 리스트와 트리 문제를 보면서, `현재 포인터가 무엇을 가리키는지`, `함수 호출 전후에 구조가 어떻게 바뀌는지`를 중심으로 이해하려고 했다.
- 수요코딩회의 SQL 처리기 구현 파트는 우선 요구사항에 따른 최소 구현 후, 학습 문서와 README를 읽으며 `main -> cli_runner -> parser -> executor -> storage` 구조에 대해 파악하였고, 이후 팀원들과 함께 작성한 세부코드를 처음부터 따라가며 흐름을 확인하였다.
- `INSERT`와 `SELECT` 를 기준으로 요청이 실제로 어떻게 흘러가는지 단계별로 따라가며 이해했다.
- CS:APP 3장도 함께 학습하려고 했지만, 실제로 읽어 보니 생각보다 밀도가 높아 시간이 많이 걸렸다. 이번 주에는 3장 전체를 끝내지는 못하였고 3.4장까지 꼼꼼하게 읽었다. 오퍼랜드 식별자에 대해 이해하였고, C 코드 실행 흐름을 더 잘 이해하게 되었다.


## 3. 문제와 해결 과정
- C언어 문법 학습 자체에 큰 어려움은 없었지만, 코드에 포인터가 등장하는 순간 변수가 지칭하는 대상이 헷갈리기 시작했다.
- 특히 자료구조 문제에서 2중 포인터가 나올 때 내가 바꾸는 것이 구체적으로 무엇인지 자주 헷갈렸다.
- 포인터가 `어떤 메모리 위치를 가리키는 변수`라는 점과, `&`는 주소를 구하는 연산, `*`는 그 주소를 따라가 값을 보는 연산이라는 점은 이해했지만, 실제 코드에 적용할 때는 여전히 헷갈렸다.
- 자료구조 문제를 풀 때 단순히 "이 문제를 풀었다"는 결과보다 "이 함수가 포인터를 어떻게 갱신하는가"를 더 이해하려고 노력했다. 연결 리스트처럼 노드 간 연결이 바뀌는 문제에서 포인터 갱신 순서를 잘못 잡으면 바로 구조가 깨지고 버그가 발생한다는 점을 체감했다.
- 이번 주에 자료구조를 완전히 자유롭게 구현할 수 있는 수준까지 갔다고 보기는 어렵다. 사전에 제공된 틀 안에서 일부 함수를 구현하고, 문제를 풀면서 C 문법과 포인터 감각을 익히는 쪽에 더 가까웠다.
- SQL 처리기에서는 입력을 읽고, 문자열을 파싱해 구조화된 명령으로 바꾸고, 실행 단계에서 분기하고, 마지막에 파일에 반영하는 흐름을 배울 수 있었다.
- `INSERT`와 `SELECT`를 각각 따라가 보면서, Parser가 SQL 문장을 `SqlCommand` 같은 내부 구조로 바꾸고, Executor가 이를 해석해 Storage를 호출하는 과정에서 책임에 따라 계층을 분리하는 패턴을 배울 수 있었다.
- 이번 주에는 디버깅이나 메모리 관리 기법 자체를 깊게 파고들지는 못했지만, 포인터와 자료구조 문제를 보면서 `malloc/free`가 왜 필요한지, 왜 노드와 같은 구조는 런타임에 동적으로 다뤄야 하는지를 배울 수 있었다.


## 4. 새로 배운 점
- C언어에서는 변수와 함수만 알아서는 부족하고, 포인터를 통해 `원본 데이터가 실제로 어떻게 바뀌는지`를 이해해야 한다.
- 포인터는 "지금 이 포인터는 무엇을 가리키는가"를 추적하는 것이 필요하다.
- 제공된 자료구조 문제에서 빈 함수를 채우며 C언어 문법, 포인터 갱신 흐름, 자료구조의 기본 동작 원리를 구체적으로 살펴볼 수 있었다.
- `Linux/gcc 사용법`은 이번 주 학습의 중심은 아니었다. 다만 C언어 과제를 진행할 때는 실행 환경과 컴파일 환경을 맞추는 것이 중요하다는 점을 확인할 수 있었다.
- `malloc/free`는 메모리 할당에서 필요하며, "필요한 메모리를 실행 중에 만들고, 더 이상 필요 없으면 정리해야 한다"는 개념을 실제 코드와 연결해서 볼 수 있었다.
- SQL 처리기를 구현하며 문자열 배열이나 2차원 형태의 데이터를 다루는 과정에서 포인터 단계가 생각보다 더 복잡해질 수 있다는 점을 배웠다.
