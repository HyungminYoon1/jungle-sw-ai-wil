# 문제
5. 페이지 폴트(Page Fault)의 개념을 설명하고, 페이지 폴트가 발생했을 때 운영체제가 수행하는 절차를 본인이 이해한 만큼 자세히 서술하시오.

---

# 답변
페이지 폴트가 무엇인지 그 개념이 확실하지는 않으나 제가 이해하고 있는 범위 내에서 운영체제의 동작을 설명하겠습니다.

운영체제는 각 프로세스의 가상메모리와 물리메모리의 페이지를 매핑하고 MMU에서는 매핑된 정보에 따라 실제 물리 메모리에 저장된 값을 읽습니다.

만약 물리 메모리의 일부 비트가 제대로 동작하지 않는 문제가 발생할 경우  RAM 드라이버는 오류가 있다는 데이터를 보내고, 운영체제는 해당 오동작 비트가 있는 물리 메모리 페이지를 향후 가상 메모리 페이지와의 매핑에 사용하지 않도록 조치합니다. 

만약 오류가 있는 물리 메모리 페이지가 이미 가상 메모리 페이지와 매핑된 상태라면 오류가 뜰 가능성이 있고, 그 경우 운영체제는 오류 메시지를 출력하고 프로세스를 중지시킵니다.

물리 메모리 페이지의 비트에 오류가 없더라도, 만약 모종의 이유로 가상 메모리 페이지와 매핑된 물리 메모리 페이지를 불러올 수 없다면 운영체제는 오류 메시지를 출력하고 프로세스를 중지시킵니다.

# 정답
페이지 폴트(Page Fault)는 프로그램이 어떤 가상 주소에 접근했을 때, 그 주소가 속한 가상 페이지를 현재 바로 사용할 수 없어서 운영체제의 처리가 필요한 상황을 말한다.

조금 더 정확히 말하면, 프로그램이 접근한 가상 페이지의 내용이 아직 물리 메모리(RAM)에 올라와 있지 않거나, 접근 권한이 맞지 않을 때 페이지 폴트가 발생한다.

여기서 중요한 점은 "가상 페이지가 아예 없는 것"이라고 단순하게 이해하면 안 된다는 것이다.  
가상 페이지는 프로세스의 가상 주소 공간에는 존재할 수 있다. 하지만 그 페이지의 내용이 아직 RAM에 적재되지 않았을 수 있고, 페이지 테이블에서도 아직 현재 사용할 수 있는 물리 프레임과 유효하게 연결되어 있지 않을 수 있다. 이런 상황에서 CPU가 그 페이지에 접근하면 페이지 폴트가 발생한다.

운영체제는 페이지 폴트가 발생하면 보통 다음 순서로 처리한다..

1. CPU가 페이지 폴트 예외를 발생시킨다.
2. 운영체제가 페이지 폴트 핸들러로 제어를 넘겨받는다.
3. 접근한 주소가 정상적인 주소인지, 그리고 읽기/쓰기 권한이 맞는지 확인한다.
4. 주소가 유효하다면, 필요한 페이지가 디스크에 저장되어 있는지 확인한다.
5. 빈 물리 메모리 프레임을 찾는다. 빈 프레임이 없으면 다른 페이지를 내보내고 자리를 만든다.
6. 디스크에 있던 필요한 페이지를 RAM으로 읽어 온다.
7. 페이지 테이블을 갱신해서 해당 가상 페이지가 이제 물리 메모리에 올라와 있음을 표시한다.
8. 중단되었던 명령어를 다시 실행한다.

만약 접근한 주소 자체가 잘못되었거나, 권한이 없는 메모리에 접근한 경우라면 운영체제는 해당 페이지를 메모리로 가져오지 않고 프로세스를 종료시키거나 예외를 전달한다.

즉, 페이지 폴트는 항상 "오류"만 의미하는 것은 아니다.  
정상적인 가상 메모리 시스템에서는 아직 RAM에 없는 페이지를 처음 사용할 때도 페이지 폴트가 발생할 수 있다. 이런 경우 운영체제가 해당 페이지를 메모리로 가져온 뒤 프로그램을 다시 실행하므로, 프로그램은 계속 정상적으로 동작할 수 있다.
