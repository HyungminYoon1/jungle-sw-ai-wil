# 문제
6. C 언어에서는 malloc() 함수를 이용해 동적으로 메모리를 할당받을 수 있다. 그렇다면 굳이 mmap() 시스템 호출을 사용하여 메모리를 할당받는 이유는 무엇인지 설명하시오.

---

# 답변
더 적은 메모리를 사용하여 할당할 수 있기 때문입니다.

---

# 정답
mmap()은 운영체제로부터 메모리 영역을 직접 매핑받기 위해 사용한다.

malloc()은 일반적으로 힙(heap) 안에서 필요한 크기만큼 메모리를 나누어 주는 고수준 할당 함수이고,
mmap()은 페이지 단위로 새로운 메모리 영역을 직접 만들거나 파일을 메모리에 매핑할 수 있는 시스템 호출이다.

따라서 mmap()을 사용하는 이유는 다음과 같다.

1. 큰 메모리 영역을 운영체제로부터 직접 할당받을 수 있다.
2. 파일 내용을 메모리에 바로 매핑할 수 있다.
3. 여러 프로세스가 같은 메모리 영역을 공유할 수 있다.
4. munmap()으로 해당 영역을 운영체제에 바로 반납할 수 있다.
5. 페이지 단위의 보호 권한(read/write/execute)을 설정할 수 있다.

즉, malloc()은 일반적인 동적 메모리 할당에 편리하고,
mmap()은 파일 매핑, 공유 메모리, 큰 메모리 영역 관리처럼 더 직접적인 제어가 필요할 때 사용한다.
