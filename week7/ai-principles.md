## 핵심 원칙

- AI는 학습 파트너이며, 결과보다 이해·검증·설명 가능 여부가 더 중요함
- 퀴즈나 제한 환경 과제에서는 AI와 인터넷을 사용하지 않음
- Malloc Lab에서 AI를 사용할 때는 `mm_init`, `mm_malloc`, `mm_free`, `mm_realloc`의 흐름과 free block 관리 방식을 직접 설명할 수 있어야 함
- 수요 코딩회에서는 AI를 적극 활용하되, B+ Tree 탐색/삽입, 자동 ID 부여, `WHERE id = ?` 조회 경로, 성능 비교 방식은 반드시 팀원이 이해하고 설명할 수 있어야 함
- AI가 제안한 구현은 바로 제출하거나 머지하지 않고, 테스트와 예제 입력으로 검증한 뒤 사용하기
- README 발표 기준으로 핵심 로직, 테스트 결과, 한계와 향후 보완점을 정리하기

## 허용하지 않는 것

- allocator 핵심 구현을 이해 없이 그대로 복붙하기
- B+ Tree 핵심 로직을 검증 없이 그대로 적용하기
- 테스트를 생략한 상태로 AI 생성 코드를 바로 제출하기
