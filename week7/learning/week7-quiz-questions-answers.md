# Week 7 쪽지시험 예상문제 20선

> Week 7 학습용 문서
>
> 목표: 7주차 쪽지시험 범위로 보이는 `학습 키워드`, `Malloc Lab`, `C 언어 코딩` 관련 예상문제 20개와 짧은 답변을 통해 빠르게 복습한다.

---

## 1. 데이터 세그먼트, 스택, 힙의 차이는 무엇인가?

### 답변

- 데이터 세그먼트: 전역 변수와 `static` 변수가 저장되는 영역
- 스택: 함수 호출, 지역 변수, 반환 주소 등이 저장되는 영역
- 힙: `malloc` 같은 동적 메모리 할당으로 얻는 영역

### 헷갈리기 쉬운 포인트

`malloc`으로 받은 포인터 변수 자체는 지역 변수이면 스택에 있고,  
그 포인터가 가리키는 실제 메모리는 힙에 있다.

---

## 2. 아래 변수들은 각각 어느 영역에 저장되는가?

```c
int g = 1;
static int s = 2;

void foo(void) {
    int x = 3;
    int *p = malloc(sizeof(int) * 4);
}
```

### 답변

- `g`: 데이터 세그먼트
- `s`: 데이터 세그먼트
- `x`: 스택
- `p`: 스택
- `malloc(...)`으로 확보한 공간: 힙

---

## 3. `malloc`과 `free`는 왜 필요한가?

### 답변

프로그램 실행 중에 필요한 크기의 메모리를 직접 요청하고 반납하기 위해 필요하다.  
즉, 컴파일 시점이 아니라 런타임에 크기가 정해지는 데이터를 다루기 위해 필요하다.

---

## 4. 시스템 콜이란 무엇인가?

### 답변

시스템 콜은 사용자 프로그램이 운영체제의 기능을 요청할 때 사용하는 인터페이스다.  
예를 들어 파일 열기, 메모리 요청, 프로세스 생성 같은 작업은 운영체제 권한이 필요하므로 시스템 콜을 통해 수행한다.

---

## 5. `sbrk`는 무엇을 하는 함수인가?

### 답변

`sbrk`는 프로세스의 힙 끝(brk)을 늘리거나 줄여서 힙 영역 크기를 조정하는 데 쓰이는 함수다.  
Malloc Lab에서는 보통 `mem_sbrk` 같은 교육용 함수로 단순화해서 다룬다.

---

## 6. `mmap`은 무엇이며 `sbrk`와 어떻게 다른가?

### 답변

`mmap`은 파일이나 익명 메모리를 가상 주소 공간에 매핑하는 방식이다.  
`sbrk`는 힙 끝을 늘리는 방식이고, `mmap`은 꼭 힙 끝이 아니어도 별도 메모리 영역을 매핑할 수 있다.

### 헷갈리기 쉬운 포인트

입문 단계에서는 `sbrk`는 힙 확장, `mmap`은 별도 매핑 방식 정도로 구분하면 충분하다.

---

## 7. 내부 단편화와 외부 단편화의 차이는 무엇인가?

### 답변

- 내부 단편화: 할당한 블록 내부에 사용하지 못하고 남는 공간
- 외부 단편화: 전체 빈 공간 합은 충분하지만, 빈 공간이 조각나 있어서 큰 블록을 할당하지 못하는 상태

---

## 8. 정렬(alignment)은 왜 필요한가?

### 답변

메모리 접근 효율과 하드웨어 제약 때문이다.  
정렬을 맞추면 CPU가 데이터를 더 효율적으로 읽고 쓸 수 있고, 어떤 시스템에서는 잘못된 정렬이 오류를 일으킬 수도 있다.

---

## 9. `implicit free list`는 무엇인가?

### 답변

힙에 있는 블록들을 앞에서부터 순서대로 훑으면서 free block을 찾는 방식이다.  
별도의 free block 전용 연결 구조 없이, 힙 전체를 순차 탐색한다.

---

## 10. `implicit free list`의 장단점은 무엇인가?

### 답변

- 장점: 구조가 단순하고 구현이 쉽다
- 단점: free block을 찾기 위해 많은 블록을 순차 탐색해야 해서 느릴 수 있다

---

## 11. `split`은 왜 필요한가?

### 답변

큰 free block을 그대로 다 써 버리면 낭비가 커질 수 있기 때문이다.  
필요한 크기만 할당하고 남는 부분을 다시 free block으로 남겨 두기 위해 split이 필요하다.

---

## 12. `coalesce`는 왜 필요한가?

### 답변

인접한 free block들을 합쳐 더 큰 free block을 만들기 위해 필요하다.  
이 과정을 통해 외부 단편화를 줄이고, 나중에 큰 메모리 요청을 처리하기 쉬워진다.

---

## 13. `mm_init`의 역할은 무엇인가?

### 답변

allocator가 동작할 수 있도록 초기 힙 구조를 준비하는 함수다.  
보통 프롤로그/에필로그를 만들고, 초기 free block을 준비하는 역할을 한다.

---

## 14. `mm_malloc`의 큰 흐름을 설명하시오.

### 답변

보통 아래 순서로 동작한다.

1. 요청 크기를 정렬 단위에 맞게 조정한다
2. 적절한 free block을 찾는다
3. 찾으면 필요 시 split 후 할당한다
4. 못 찾으면 힙을 확장한다
5. 새 공간에서 다시 할당한다

---

## 15. `mm_free`가 호출되면 allocator 내부에서 어떤 일이 일어나는가?

### 답변

할당된 블록을 free 상태로 표시하고,  
가능하면 앞뒤 인접 free block과 병합(coalesce)한다.

---

## 16. `mm_realloc`은 왜 단순한 `free`와 같지 않은가?

### 답변

`realloc`은 기존 데이터를 유지하면서 크기를 바꾸는 함수이기 때문이다.  
필요하면 새 블록을 할당하고 데이터를 복사한 뒤, 기존 블록을 반납해야 할 수도 있다.

---

## 17. 힙 공간이 부족하면 allocator는 보통 어떻게 동작하는가?

### 답변

적절한 free block을 찾지 못하면 힙 자체를 더 확장하려고 시도한다.  
Malloc Lab에서는 보통 `mem_sbrk` 같은 함수를 통해 이 과정을 단순화해서 구현한다.

---

## 18. `malloc`이 반환하는 포인터는 보통 무엇을 가리키는가?

### 답변

보통 사용자가 실제로 쓰는 payload의 시작 주소를 가리킨다.  
header/footer 같은 allocator 내부 메타데이터를 가리키는 것이 아니다.

---

## 19. 메모리 누수(memory leak)란 무엇인가?

### 답변

더 이상 쓰지 않는 동적 메모리를 `free`하지 않아서, 프로그램이 그 메모리를 계속 잃어버린 상태를 말한다.

예:

```c
void foo(void) {
    int *p = malloc(sizeof(int) * 100);
}
```

위 코드는 `p`를 `free`하지 않으므로 누수가 발생한다.

---

## 20. 아래 코드는 왜 위험한가?

```c
int *p = malloc(sizeof(int) * 10);
p[0] = 1;
free(p);
printf("%d\n", p[0]);
```

### 답변

`free(p)` 이후의 메모리는 더 이상 유효한 사용 대상이 아니기 때문이다.  
즉, 해제된 메모리에 다시 접근하는 use-after-free 문제다.
