## 6. 기타 협업 팁

- Malloc Lab은 점수만 올리는 것보다 allocator 흐름을 설명할 수 있는지 먼저 확인하기
- 수요 코딩회는 기존 SQL 처리기와의 연동이 핵심이므로, 이전 구조를 먼저 공유한 뒤 B+ Tree 추가 범위를 정하기
- 테스트는 마지막에 몰아서 하지 말고, parser/insert/select/index 경로별로 중간 검증하기
- README 발표 기준으로 구현 목적, 인덱스 경로, 성능 비교 결과, 한계를 함께 정리하기
- AI 생성 코드 사용 시 바로 머지하지 않고, 포인터 흐름과 자료구조 변경 과정을 직접 확인한 뒤 팀원에게 설명하기
