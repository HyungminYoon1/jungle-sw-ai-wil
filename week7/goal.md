# Week 7 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** 핵심 주제 8개(`Malloc Lab 환경 이해`, `implicit list`, `mm_init`, `mm_malloc`, `mm_free`, `mm_realloc`, `B+ Tree 인덱스 흐름`, `WHERE id = ? 조회 흐름`)를 학습하고 적용해 총 **8개 이상 문제 해결**
- **달성률 산식:** (해결한 핵심 주제 수 / 8) × 100


## 2. 설계
- **목표:** 수요 코딩회 B+ Tree 인덱스 구현 시 **자동 ID 부여, 인덱스 등록 구조, `WHERE id = ?` 조회 경로, 선형 탐색 비교 경로, 대량 데이터 테스트 방식** 등 설계 포인트 **5개 이상**을 README 또는 설계 메모에 명시
- **달성률 산식:** (문서에 명시한 설계 포인트 수 / 5) × 100


## 3. 구현
- **목표:** Malloc Lab 핵심 함수 4개(`mm_init`, `mm_malloc`, `mm_free`, `mm_realloc`) + SQL 처리기 확장 핵심 기능 6개(`자동 ID`, `B+ Tree 삽입`, `B+ Tree 검색`, `WHERE id = ?`, 선형 탐색 비교, 대량 데이터 삽입`) = **10개 이상 구현 완료**
- **달성률 산식:** (완료한 구현 항목 수 / 10) × 100


## 4. 품질
- **목표:** `mdriver` 채점 1개 + Malloc Lab 개별 검증 1개 + B+ Tree 단위 테스트 2개 + SQL 처리기 기능 테스트 2개 + 성능 비교 테스트 1개 = **7개 이상 검증 항목 통과**
- **달성률 산식:** (통과한 검증 항목 수 / 7) × 100


## 5. 유지보수
- **목표:** 이번 주(금~목) **7일 중 5일 이상** 개인 또는 팀 레포지토리에 의미 있는 커밋 메시지로 Malloc Lab, 메모리 학습, B+ Tree 인덱스, 테스트, README 작업을 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 팀 협업 룰/팁 합의(1개) + Malloc Lab 개념 공유(1개) + B+ Tree 인덱스 팀 프로젝트 참여(1개) = **3개 완료**
- **달성률 산식:** (완료한 항목 수 / 3) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 이번 주에 구현하는 B+ Tree 인덱스가 **왜 필요한지**, 그리고 선형 탐색과 어떤 차이가 있는지를 README에 **2문장 이상** 명시
- **달성률 산식:** (명시한 문장 수 / 2) × 100


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + allocator 또는 B+ Tree 핵심 로직 검증·원리 설명(1개) = **2개 완료**
