# Week 3 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.  
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** basic 9개 + 난이도 하 6개 + 난이도 중 6개 = 총 21개 과제 중 **21개 해결**
- **달성률 산식:** (해결한 과제 수 / 21) × 100


## 2. 설계
- **목표:** 수요 코딩회 미니 Redis 구현 시 **해시 테이블 설계 원리·아키텍처를 문서화** 후 구현. 동시성, TTL, API 구조 등 설계 포인트 **3개 이상** 문서에 명시
- **달성률 산식:** (문서에 명시한 설계 포인트 수 / 3) × 100


## 3. 구현
- **목표:** basic 9개 테스트 통과 + 난이도 하·중 12개 AC = **21개 구현 완료**
- **달성률 산식:** (테스트 통과 + AC 완료한 문제 수 / 21) × 100


## 4. 품질
- **목표:** 난이도 하·중 12개 백준 AC 통과 + 미니 Redis **단위 테스트·기능 테스트** 작성 및 통과 = **13개 항목**
- **달성률 산식:** (통과한 항목 수 / 13) × 100


## 5. 유지보수
- **목표:** 이번 주(금~목) **7일 중 5일 이상** 개인 레포지토리에 의미 있는 커밋 메시지로 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 팀 협업 룰/팁 합의(1개) + 수요 코딩회 팀 활동 참여(1개) + 코어타임 코딩테스트 3문제 이상 팀 풀이(1개) = **3개 완료**
- **달성률 산식:** (완료한 항목 수 / 3) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 미니 Redis의 **실제 활용 시나리오**(캐시, 세션 등)를 README에 1문장 이상 명시
- **달성률 산식:** 명시 여부 (0% / 100%)


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + 미니 Redis에서 AI 생성 코드 검증·원리 설명(발표 시 1회 이상) = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 10. 학습 민첩성
- **목표:** basic 9개 주제(이분탐색, 분할정복, 퀵 정렬, 머지 정렬, 스택, 큐, 우선순위 큐, 링크드리스트, 해시 테이블) 중 **9개 모두** 학습 후 적용
- **달성률 산식:** (학습 후 적용 완료한 주제 수 / 9) × 100


---

### 달성률 산식 참고
- 문제해결/구현: 과제·문제 단위로 카운트
- 품질: 백준 AC + 미니 Redis 테스트 항목
- 유지보수: 일 단위 커밋 빈도
- 협업/태도/AI 활용: 완료 항목 수
- 비즈니스 이해: 명시 여부 (예/아니오)
