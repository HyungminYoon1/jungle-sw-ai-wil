# Redis 학습 정리

## 1. Redis란 무엇인가

Redis는 Remote Dictionary Server의 약자로, 메모리 기반의 키-값 저장소이다. 단순한 문자열 저장소를 넘어 다양한 자료구조를 제공하며, 빠른 읽기/쓰기가 필요한 상황에서 자주 사용된다.

Redis는 일반적인 관계형 데이터베이스처럼 복잡한 조인이나 정규화를 중심으로 사용하기보다, 짧은 응답 시간과 단순한 데이터 접근이 중요한 문제를 해결하는 데 적합하다.

## 2. Redis의 핵심 특징

### 메모리 기반 저장
Redis는 데이터를 주로 메모리에 저장하므로 디스크 기반 데이터베이스보다 매우 빠른 응답 속도를 기대할 수 있다. 대신 메모리는 비용이 높고 용량이 제한적이므로 모든 데이터를 Redis에 저장하는 것은 일반적이지 않다.

### 키-값 구조
기본적으로 `key -> value` 형태로 데이터를 저장한다. 키를 통해 곧바로 값에 접근할 수 있기 때문에 조회 경로가 단순하고 빠르다.

### 다양한 자료구조 지원
Redis는 문자열 하나만 저장하는 저장소가 아니라 다음과 같은 자료구조를 제공한다.

- String
- List
- Set
- Hash
- Sorted Set
- Bitmap
- HyperLogLog
- Stream

이 덕분에 단순 캐시뿐 아니라 순위표, 작업 큐, 세션 저장소 같은 기능도 비교적 자연스럽게 구현할 수 있다.

### TTL 지원
Redis는 키마다 만료 시간(Time To Live)을 설정할 수 있다. 이 기능은 캐시 데이터, 인증 코드, 세션처럼 일정 시간이 지나면 자동으로 삭제되어야 하는 데이터를 다룰 때 매우 유용하다.

## 3. Redis가 빠른 이유

Redis가 빠른 가장 큰 이유는 다음과 같다.

- 메모리 기반으로 동작해 디스크 I/O 비용이 작다.
- 데이터 접근 방식이 단순한 키 조회 중심이다.
- 내부 자료구조가 특정 연산에 최적화되어 있다.
- 단일 스레드 이벤트 루프 기반으로 동작하는 구조 덕분에 락 경쟁이 상대적으로 적다.

물론 실제 성능은 네트워크 환경, 데이터 크기, 명령 종류, 운영 방식에 따라 달라질 수 있다.

## 4. Redis의 장점

- 매우 빠른 읽기/쓰기 성능
- TTL을 이용한 자동 만료 처리
- 다양한 자료구조 지원
- 구현과 사용 방식이 비교적 단순함
- 캐시, 세션, 카운터, 큐 등 여러 문제에 폭넓게 적용 가능

## 5. Redis의 한계

- 메모리 사용량이 크기 때문에 대용량 데이터를 모두 저장하기에는 비용 부담이 있다.
- 관계형 데이터베이스처럼 복잡한 질의에 적합하지 않다.
- 영속성을 지원하더라도 기본 목적은 빠른 인메모리 처리이므로, 핵심 원본 데이터를 완전히 Redis에만 의존하는 설계는 주의가 필요하다.
- 분산 환경이나 장애 복구까지 고려하면 운영 복잡도가 올라간다.

## 6. 언제 Redis를 사용하는가

Redis는 다음과 같은 상황에서 특히 적합하다.

- 같은 데이터를 자주 읽어 데이터베이스 부하를 줄이고 싶을 때
- 짧은 수명의 데이터를 빠르게 저장하고 삭제하고 싶을 때
- 실시간 카운터나 순위 계산이 필요할 때
- 세션 상태를 빠르게 조회해야 할 때
- 큐 또는 이벤트 전달 기능이 필요할 때

## 7. 학습 관점에서 중요한 포인트

이번 주차에서 Redis를 학습할 때는 다음 요소를 중심으로 이해하면 좋다.

- 왜 메모리 기반 저장소가 필요한가
- 왜 TTL이 캐시와 세션 문제를 풀기에 적합한가
- 키-값 저장소가 관계형 데이터베이스와 어떻게 역할이 다른가
- Redis의 다양한 자료구조가 실제 문제와 어떻게 연결되는가
- 미니 Redis를 구현할 때 어떤 기능부터 단순하게 시작할 수 있는가

## 8. 미니 Redis 구현으로 연결되는 개념

학습용 미니 Redis를 만든다면 처음부터 Redis의 모든 자료구조를 지원할 필요는 없다. 아래와 같은 최소 기능부터 시작하는 것이 적절하다.

- 문자열 기반 키-값 저장
- `SET`, `GET`, `DEL` 같은 기본 명령
- TTL 설정과 만료 처리
- 간단한 메모리 저장소 구조

이 정도만 구현해도 Redis의 핵심 개념인 키-값 저장, 빠른 조회, 만료 처리 방식을 충분히 학습할 수 있다.
