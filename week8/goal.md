# Week 8 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** 핵심 주제 8개(`BSD socket`, `IP/TCP/HTTP`, `file descriptor`, `DNS`, `echo/client-server`, `tiny 웹서버`, `proxy 기본 흐름`, `미니 DBMS API 서버`)를 학습하고 적용해 총 **8개 이상 문제 해결**
- **달성률 산식:** (해결한 핵심 주제 수 / 8) × 100


## 2. 설계
- **목표:** 수요 코딩회 API 서버 구현 시 **스레드 풀 구조, 요청 처리 흐름, DB 엔진 연결 방식, 동시성 대응 포인트, 테스트 방식** 등 설계 포인트 **5개 이상**을 README 또는 설계 메모에 명시
- **달성률 산식:** (문서에 명시한 설계 포인트 수 / 5) × 100


## 3. 구현
- **목표:** 웹/프록시 서버 핵심 구현 5개(`socket 기반 요청/응답`, `echo/client-server`, `tiny 웹서버`, `homework 3문제 이상`, `proxy 과제 도전`) + API 서버 핵심 구현 4개(`외부 API 요청 처리`, `스레드 풀`, `병렬 SQL 처리`, `DB 엔진 연동`) = **9개 이상 구현 완료**
- **달성률 산식:** (완료한 구현 항목 수 / 9) × 100


## 4. 품질
- **목표:** tiny 웹서버 동작 확인 1개 + homework 풀이 검토 1개 + API 서버 단위 테스트 2개 + API 기능 테스트 2개 + README 데모 점검 1개 = **7개 이상 검증 항목 통과**
- **달성률 산식:** (통과한 검증 항목 수 / 7) × 100


## 5. 유지보수
- **목표:** 이번 주(금~목) **7일 중 5일 이상** 개인 또는 팀 레포지토리에 의미 있는 커밋 메시지로 웹/프록시 서버, API 서버, 테스트, README 작업을 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 팀 협업 룰 정리(1개) + 웹/프록시 서버 학습 공유(1개) + API 서버 팀 프로젝트 참여(1개) = **3개 완료**
- **달성률 산식:** (완료한 항목 수 / 3) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 이번 주 API 서버와 프록시 서버가 **왜 필요한지**, 그리고 외부 요청 처리와 내부 DB 엔진 연결이 어떤 의미를 가지는지 README에 **2문장 이상** 명시
- **달성률 산식:** (명시한 문장 수 / 2) × 100


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + 웹/프록시 또는 API 서버 핵심 로직 검증·원리 설명(1개) = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 10. 학습 민첩성
- **목표:** `BSD socket`, `IP`, `TCP`, `HTTP`, `file descriptor`, `DNS`, `tiny`, `proxy`, `thread pool`, `동시성 이슈`, `API 서버 아키텍처`를 포함해 **11개 이상 개념/구현 포인트**를 학습 후 적용
- **달성률 산식:** (학습 후 적용 완료한 개념/구현 포인트 수 / 11) × 100
