## 핵심 원칙

- AI는 구현 속도를 높이는 도구로 사용하되, 핵심 로직은 반드시 이해하고 설명할 수 있어야 함
- 퀴즈나 제한 환경 과제에서는 AI와 인터넷을 사용하지 않음
- Web/Proxy Server 과제에서 AI를 사용할 때는 socket 생성, 요청/응답 흐름, HTTP 처리, proxy 전달 구조를 직접 설명할 수 있어야 함
- 수요 코딩회에서는 AI를 적극 활용하되, 스레드 풀, 병렬 요청 처리, 내부 DB 엔진 연동, 테스트 전략은 반드시 팀원이 이해하고 설명할 수 있어야 함
- AI가 제안한 구현은 바로 제출하거나 머지하지 않고, 예제 요청과 테스트 케이스로 검증한 뒤 사용하기
- README 발표 기준으로 핵심 구조, 테스트 결과, 동시성 이슈와 한계를 정리하기

## 허용하는 것

- BSD socket, TCP/IP, HTTP, DNS 개념 정리 요청
- tiny 웹서버와 proxy 흐름 설명 요청
- 오류 메시지 해석과 디버깅 방향 점검
- API 서버 구조화, 테스트 케이스, README 정리 보조
- 동시성 이슈 점검과 검증 포인트 정리

## 허용하지 않는 것

- socket, proxy, 스레드 풀 핵심 구현을 이해 없이 그대로 복붙하기
- 테스트를 생략한 상태로 AI 생성 코드를 바로 제출하기
- 설명할 수 없는 코드를 발표용 결과물에 포함하기
