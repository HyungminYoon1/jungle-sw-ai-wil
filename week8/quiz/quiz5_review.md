# 문제
5. 웹 브라우져에 https://www.google.com/search?q=정글 입력 후 주소창에 엔터를 치는 순간부터 화면이 보이기까지의 과정을 네트워크 계층과 시스템 관점에서 단계별로 설명 해 보시오.

# 답변

네트워크 계층
1. 응용 계층: HTTP 같은 애플리케이션 프로토콜
2. 전송 계층: TCP, UDP 같은 데이터 전달 방식
3. 인터넷 계층: IP를 통한 호스트 간 전달
4. 링크 계층: 실제 네트워크 장치를 통한 프레임 전달

=> 데이터는 송신자 프로세스의 응용 계층부터 순차적으로 하위 계층으로 내려갔다가 다시 상위 계층으로 올라가며 수신자 프로세스로 전달됩니다.

시스템 관점
-> 클라이언트 브라우저에서 URL 입력
-> DNS가 도메인 이름을 IP 주소로 변환
-> 브라우저가 서버의 IP:포트로 TCP 연결
-> 브라우저가 HTTP 요청 전송
-> 데이터(패킷)가 물리적 통신망을 통해 서버로 이동
-> 서버 프로세스가 socket으로 요청을 읽음
-> 서버가 파일 또는 내부 로직 처리
-> 서버가 HTTP 응답 전송
-> 데이터(패킷)가 물리적 통신망을 통해 클라이언트 PC로 이동
-> 클라이언트 프로세스(브라우저)가 socket으로 응답을 읽음
-> 브라우저가 결과 표시

# 정답
사용자가 브라우저에 URL을 입력하고 엔터를 치면, 브라우저는 먼저 도메인 이름(www.google.com)에 대한 DNS 조회를 수행하여 IP 주소를 얻는다. 그 후 해당 서버와 TCP 연결을 맺고, HTTPS이므로 TLS 핸드셰이크를 통해 암호화 세션을 수립한다. 연결이 완료되면 브라우저는 HTTP GET 요청을 서버에 전송한다. 서버는 요청을 처리한 뒤 HTML 응답을 반환한다. 브라우저는 이 HTML을 파싱하면서 필요한 CSS, JavaScript, 이미지 등의 추가 리소스를 다시 요청한다. 이후 DOM 생성, 레이아웃 계산, 페인팅 과정을 거쳐 최종 화면을 사용자에게 보여준다.
